FaslBook V2 - Complete Product Documentation

Vision

FaslBook is a farm operating system for landlords, managers, farmers, and workers.

The system must be workflow-driven, not screen-driven.

Every action should automatically update all connected modules.

Example:

Manager gives 10 Maund Wheat to Farmer

Automatically:

- Godown stock decreases
- Farmer stock increases
- Inventory history created
- Ledger updated
- Reports updated
- Activity log created

Users should never need to update multiple modules manually.

The database must always remain the single source of truth.

---

User Roles

Landlord

Full access.

Can:

- View all data
- Manage land
- Manage parcels
- Manage farmers
- Manage managers
- Manage workers
- Manage inventory
- Manage dealers
- Manage loans
- Manage finances
- Generate reports
- Approve join requests

Cannot be removed except by ownership transfer.

---

Manager

Operational access.

Can:

- Add expenses
- Add income
- Manage inventory
- Manage workers
- Manage attendance
- Manage crop records
- Manage dealer transactions

Cannot:

- Delete organization
- Transfer ownership
- Delete landlord

---

Farmer

Limited access.

Can:

- View assigned parcels
- View assigned crops
- View stock received
- View harvest records
- View own reports

Cannot see other farmers or organization finances.

---

Worker

Minimal access.

Can:

- View attendance
- View payments
- View assigned work

Cannot access financial records.

Core System Rules

Rule 1:

Never type names manually.

Always select existing records.

Example:

Bad:
Worker Name = Ahmed

Good:
Select Worker From Team

---

Rule 2:

Balances are never editable.

Everything is calculated.

---

Rule 3:

Ledger is system-generated only.

No manual ledger entries.

---

Rule 4:

Every action creates activity logs.

- User

- Record
